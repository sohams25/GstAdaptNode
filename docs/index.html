Why GstAdaptNode?

Hardware Auto-Detection

Probes /dev/ for NVIDIA Jetson or Intel VA-API at startup. Falls back to CPU automatically. No config files, no environment variables.

True Zero-Copy IPC

Custom appsrc wrapper moves the ROS message pointer directly into GStreamer via gst_buffer_new_wrapped_full. Zero serialization, zero memcpy.

5.6x CPU Reduction

Offloads 4K resize to VA-API/NVMM hardware. Drops from 152% CPU to 27% per stream. Freed cores for SLAM, perception, planning.

The Plug-and-Play Promise

Same parameters. Same composable node interface. Your camera still publishes bgr8. Toggle below to see the difference.

Quick Start

# Clone
cd ~/ros2_ws/src
git clone --recursive https://github.com/sohams25/GstAdaptNode.git gst_adapt_node

# Dependencies
sudo apt install ros-humble-image-proc gstreamer1.0-vaapi \
  libgstreamer1.0-dev libgstreamer-plugins-base1.0-dev

# Build
cd ~/ros2_ws && colcon build --packages-up-to gst_adapt_node
source install/setup.bash

# A/B stress test
ros2 launch gst_adapt_node A_B_comparison.launch.py video_path:=/path/to/4k.mp4

# Visual dashboard (second terminal)
ros2 run gst_adapt_node visualize_demo.py
